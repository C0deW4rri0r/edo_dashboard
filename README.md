# Аналитический дашборд ЭДО

Демонстрационный веб-прототип аналитики для системы электронного документооборота. Проект разработан как дополнительный модуль к основной работе по практике — десктопному сборщику логов. Дашборд не заменяет основной проект, а показывает возможное направление развития: визуализацию накопленных показателей по договорам, согласованиям, просрочкам и динамике работы.

## Статус проекта

Текущая версия является рабочим учебным прототипом. По умолчанию используется SQLite и сгенерированные тестовые данные. Интеграция с реальной базой ЭДО, промышленная авторизация и эксплуатационные настройки безопасности в прототип не включены.

Такой статус выбран осознанно: задача дополнительного проекта — показать архитектуру аналитического веб-модуля, API, модели данных, графики и интерфейс, а не заменить основную систему сбора логов.

## Возможности

| Блок | Реализовано |
|---|---|
| KPI-карточки | Всего договоров, на согласовании, просрочено, среднее время подписания |
| Воронка согласования | Распределение договоров по этапам жизненного цикла |
| Динамика по месяцам | Сравнение количества созданных и подписанных договоров за 12 календарных месяцев |
| Распределение по типам | Аналитика по типам договоров |
| Медленные согласования | Таблица топ-5 зависших договоров |
| API | JSON-endpoints для фронтенда |
| Тестовые данные | Команда генерации демонстрационной выборки |
| Docker | Быстрый запуск через Docker Compose |

## Технологии

| Уровень | Технологии |

## Быстрый запуск локально

```bash
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
python manage.py migrate
python manage.py seed_data
python manage.py runserver
```

После запуска дашборд доступен по адресу `http://127.0.0.1:8000/`.

## Запуск через Docker

```bash
docker compose up --build
```

Docker Compose применяет миграции, генерирует тестовые данные и запускает Django-сервер на порту 8000.

## API

| Endpoint | Назначение |

| `/api/kpi/` | KPI-карточки |
| `/api/funnel/` | Воронка согласования |
| `/api/dynamics/` | Динамика по месяцам |
| `/api/types/` | Распределение по типам договоров |
| `/api/slowest/` | Топ-5 медленных согласований |

## Тестирование

```bash
python manage.py test
```

В проект добавлены smoke-тесты API. Они проверяют доступность основных endpoints, структуру JSON-ответов, расчет KPI, уникальность 12 месяцев в графике динамики и выдачу зависших договоров.

## Ограничения

| Ограничение | Комментарий |
|---|---|
| Нет связи с реальной БД ЭДО | Используется демонстрационная SQLite-база |
| Нет интеграции с LogCollectorApp | Дашборд является отдельным прототипом, а не частью основного сборщика логов |
| Нет промышленной авторизации | Для реального внедрения нужны роли пользователей и разграничение доступа |
| Внешние frontend-библиотеки подключаются через CDN | Для закрытой сети их нужно перенести в локальную статику |
| API кэшируется на 5 минут | Интерфейс обновляет данные раз в 60 секунд, но фактическое обновление API зависит от кэша |

## Формулировка для защиты

Дашборд следует представлять как дополнительный рабочий прототип. Основной результат практики — десктопная система автоматического сбора логов. Веб-дашборд демонстрирует, как в дальнейшем можно развить проект в сторону аналитики: отображать статистику по процессам ЭДО, отслеживать просрочки, видеть динамику и быстро находить проблемные участки. На текущем этапе дашборд работает на тестовых данных, но уже содержит backend, модели, API, графики, таблицу, адаптивный интерфейс и базовые автоматические тесты.
